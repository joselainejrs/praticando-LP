###  SUB-PROGRAMAS:

```
Explicação:
    - É um programa que auxilia o programa principal através da realização de uma determinada subtarefa;
    - Também costuma receber os nomes de sub-rotina, procedimentos, métodos ou módulo;
    - Os subprogramas são chamadas dentro do corpo do programa principal como se fossem comandos;
    - Após seu termino, a execução continua a partir do ponto onde foi chamado;
    - É importante compreender que a chamada de um subprograma simplesmente gera um desvio provisória no
      fluxo de execução;
    - Há um caso particular de subprograma que recebe o nome de função. Uma função. além de executar uma determinada tarefa,
     retorna um valor para quem a chamou, que é o resultado de sua execução. Por este motivo, a chamada de uma função a
     parece no corpo principal do programa como uma expressão e não como um comando;
    - Cada subprograma, pode conter dois tipos de variaveis a GLOBAL e LOCAL
      variáveis (variavéis locais) que existem apenas durante sua chamada;
    - Ao se chamar um subprograma também é possivél passar-lhe determinadas informações que recebem o nome de parâmentros
      (são valores que, na linha de chamada, ficam entre os parênteses e que são separados por vírgulas). A quantidade
      de parâmetros, sua sequência e respectivos tipos não podem mudar, devem estar de acordo como o que foi especificado
      na sua correspondente declaração;
    - Função devolve um resposta
    
    Resumindo a explicação:
      São conhecido como função ou método;

      subString (copia)
    
    Ex: 
      -O que é estritamente necessário saber de uma função:
      nome
      objeto
      Parâmentros de entrada (ordem)
      tipo de resposta
        

```

###  Variaveis Locais e Global:    

```  
Explicação:

  - Variáveis global: Ela fica aramazenado na memoria em todo momento do projeto; 
  - Variaveis Local: Ela fica armazenado na memoria durante a execução especifica de alguma função ou procedimento;
         
```
